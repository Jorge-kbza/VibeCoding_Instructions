## OBJETIVO

Construir una API REST que exponga un endpoint:

GET /weather

---

## REQUISITOS FUNCIONALES

- Consumir datos meteorológicos desde:
https://api.open-meteo.com/v1/forecast?latitude=39.47&longitude=-0.38&current_weather=true

- Extraer de la respuesta:
  - temperatura
  - velocidad del viento

- Transformar los datos para devolver:

Temperatura:
- Celsius
- Fahrenheit

Velocidad del viento:
- km/h
- m/s

---

## FORMATO DE RESPUESTA

La API debe responder exactamente con esta estructura:

{
  "temperature": {
    "celsius": number,
    "fahrenheit": number
  },
  "wind_speed": {
    "kmh": number,
    "ms": number
  }
}

---

## REQUISITOS TÉCNICOS

- Usar Node.js con Express
- Seguir arquitectura MVC estricta
- Separar correctamente:
  - controllers
  - services
  - utils
  - routes
  - middlewares

---

## REGLAS IMPORTANTES

- NO poner lógica de negocio en controllers
- Las conversiones deben estar en utils como funciones puras
- El consumo de la API externa debe estar en services
- Usar async/await
- Manejar errores usando middleware global (según instructions)
- Validar la respuesta de la API externa

---

## TESTING

- Crear tests unitarios usando Jest
- Testear obligatoriamente:
  - conversión de temperatura
  - conversión de velocidad
